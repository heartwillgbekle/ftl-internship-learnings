## HTML and CSS Review

### HTML: `href` vs `src`

**`href` — "hypertext reference"**
- Goes to an external location
- Used for links
- Example: `<a href="https://example.com">Link</a>`

**`src` — "source"**
- Grabs the content and puts it in the page
- Used for images, audio, source code
- Example: `<img src="image.png">`, `<script src="app.js">`

### CSS: Flexbox Highlights

**Parent Container Attributes:**
- Parent has `display: flex;`
- `flex-direction` set to `row` or `column`
- `flex-wrap` describes whether items "wrap" onto the next row
- `justify-content` for how children are spaced
- `gap` on parent for spacing between children

**Child Attributes:**
- Child might specify `margin` for individual spacing

### CSS: Media Queries Review

**Structure:**
- `@media` keyword followed by the query and a rule
- The rule is conditionally applied if the query is true
  - Example: `(max-width: 600px)`
- You can add type (like `screen`) and logical operators

**Example:**
```css
@media screen and (max-width: 600px) {
  /* Styles applied when screen width is 600px or less */
}
```

## Project Demos & Presentations

### Presentation Structure

**Time Allocation:**
- 4-5 minutes to present your project
- 2 minutes of Q&A and feedback from peers

**What to Cover:**
1. What does your project do?
2. What features are implemented (especially any stretch features)?
3. What did you enjoy the most while working on your project?
4. What did you find the most interesting and challenging?
5. What would you want to do next if you had more time? Would you do anything differently?

### Presentation Tips

- **Create a slide** for presenting — this will help guide your presentation and talking points
- **Be brief and concise** — focus on the main points in your project and what makes your project unique
- **Practice makes improvement** — we don't expect perfection! The more you practice speaking, the easier it will be for future demonstrations

### Presentation Slide Template

**Slide 1: Overview**
- Project name and description
- What does your project do?

**Slide 2: Favorite Feature**
- Feature information
- Why you enjoyed implementing it

**Slide 3: Most Challenging**
- Feature information
- What made it challenging and how you overcame it

**Slide 4: Next Steps**
- What you would do next
- Things you'd do differently

**Slide 5: Acknowledgments**
- Shout out to people who helped
- Resources used

### Glows and Grows Feedback Framework

**Glows:**
- What did they do well in their presentation?
- What features did you like about their projects?

**Grows:**
- What suggestions do you have on how they can improve in their project or presentation?

**During Q&A:**
- Feel free to ask the presenter questions about how they implemented certain features or how they planned their project

## Key Takeaways

- **`href`** is for linking to external locations; **`src`** is for embedding content
- Flexbox parent controls the layout of children with `display: flex;`, `flex-direction`, and `justify-content`
- Media queries use `@media` to conditionally apply CSS based on device characteristics
- Effective presentations focus on what makes your project unique and the challenges overcome
- **Glows and Grows** framework provides constructive feedback
- Presenting technical work is a skill that improves with practice
